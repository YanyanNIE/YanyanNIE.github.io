comment construire votre web personnel avec GitHub Pages et Hexo

Posted 2018-05-19 | Modified: 2019-11-29 | In config | | views
: 674 | ≈ 4

Je suis toujours besoin de construire mon propre web site. Après la fin de mon stage, j’ai décidé d’étudier sur ça.

Travailles de préparation:

  • Un compte de GitHub;
  • Installation de Node.js et npm;
  • Installation de git.

Environnements:

  • Mac OS 10.13.x
  • Node.js 10.0.0 node -v
  • npm 5.6.0 npm -v
  • git 2.15.1 (Apple Git-101) git version
  • hexo 3.7.1 hexo version ou hexo -v

Créer une repository

Créer une nouvelle repository avec le nom USER_NAME.github.io, où <USER_NAME> est ton nom d’utilisatuer de GitHub. Par exemple, mon nom d’utilisatuer de GitHub est de YanyanNIE, du coup il faut que je crée la repository avec le nom YanyanNIE.github.io.

Configuration de SSH KEY

Dans le terminal, taper ssh -v pour regarder la version de ssh, le résultat comme ci-dessous montre que ssh est déjà installé.

Si vous n’avez installé ssh, vous pouvez l’installer par la commande:

ssh-keygen -t rsa -C "youremail@example.com"

avec contibuer à appuyer sur Entrée

Après l’installation, vous avez un fichier .sh sous votre compte d’ordinateur,

Et votre ssh key est dans la fichier id_rsa.pub. Vous pouvez le copier et coller dans votre compte de GitHub.

Créer votre site avec Hexo

Installation de Hexo

Utiliser la commande ci-dessous pour installer Hexo globalement:

npm install -g hexo

Initialisation

Ensuite, il faut que vous créez une répertoire pour stocker des codes de votre blog. Je vous conseille de la mettre un lieu avec prudent. Par exemple, dans mon ordinateur, je veux le mettre dans ~/myGitHub/blog, alors il y a deux manières de faire l’initialisation:

  • sous le répertoire ~/myGitHub, j’execute hexo init blog
  • sous le répertoire ~/myGitHub, créer une répertoire mkdir blog, puis change sous la répertoire blog et puis hexo init.

Il faut faire attention que la répertoire doit être vide, sinon, il y aura des erreurs.

Après, sous la répertoire blog, vous pouvez voir des fichiers initialisés.

Générer le site et le regarder localement

Maintenant vous pouvez directement générer le site avec la commande hexo g, il va générer une nouvelle répertoire public sous blog. Et avec la commande hexo s, vous pouvez regarder la prévu du site localement par le lien http://localhost:4000/ (Par défault). La figrue ci-dessous est une vue ensemble du site.
